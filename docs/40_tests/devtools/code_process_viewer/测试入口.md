# 代码过程浏览可视工具测试入口

## 首版验收目标

验证该工具是否能让用户完成以下动作：

1. 导入一次真实 `jigsaw_solver_debug/<runId>`。
2. 看懂 Jigsaw 求解步骤流程。
3. 点开节点查看预览图、证据和源码入口。
4. 看懂节点产出的结构化数据表格。
5. 标记通过 / 存疑 / 驳回 / 待补证据。
6. 导出一份可回看的审查结果。

## 当前部署约定

- 首版默认访问端口：`6657`
- 首版默认访问方式：同机或局域网浏览器访问

## 首版测试清单

| 编号 | 测试项 | 预期结果 |
| --- | --- | --- |
| T1 | 导入真实调试目录 | 成功生成 1 个 Session，且 `debug_run_id` 正确 |
| T2 | 打开会话列表 | 能看到会话名、状态、时间、项目 |
| T3 | 打开单个会话图 | 能正确展示节点、边、状态颜色 |
| T4 | 点击节点 | 右侧详情面板展示目标、结果、源码引用 |
| T5 | 查看预览图 | 有图步骤可展示，缺图步骤明确提示证据不完整 |
| T6 | 查看表格 | `evidence` / `legend` 能展示结构化列和行 |
| T7 | 标记审查结果 | 能保存通过、存疑、驳回、待补证据并刷新保留 |
| T8 | 查看关键步骤 | 能逐步切换 `request_context` / `runtime_jigsaws` / `parent_connector_resolution` / `vanilla_piece_result` / `validation_result` / `apply_result` |
| T9 | 导出报告 | 能导出 Markdown 报告 |
| T10 | 局域网访问 | 同网段设备能正常访问页面和接口 |

## 非功能验证

| 项目 | 验证点 |
| --- | --- |
| 部署 | 单机启动后前后端都可访问 |
| 安全 | 首版至少确保只读导入，不暴露求解触发能力 |
| 性能 | 单次 Jigsaw 调试会话能在可接受时间内打开 |
| 可回看 | 历史会话可再次打开，不因新会话覆盖 |

## 首版建议证据

- 一份真实 `jigsaw_solver_debug/<runId>`
- 一份缺少预览图的步骤样本
- 一份带人工审查标记的导出报告
