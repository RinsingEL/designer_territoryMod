# 影响面

## 首版代码过程浏览工作台

### 影响的模块、接口、配置、阶段

- 新增独立系统 `code_process_viewer`
- 影响产品方案、展示模型、代码导览、测试入口的整套文档骨架
- 后续实现时会新增 Go 服务、前端静态资源、会话存储、审查记录与导出能力
- 首版默认固定使用 7 类检查点节点和开发端口 `6657`

### 影响的测试入口或验收口径

- 新增“会话图是否可读”“证据是否可追溯”“审查标记是否可保存”“多次结果是否可对比”等验收点
- 局域网访问能力与基础鉴权需要纳入验收

### 影响的上下游系统

- 上游依赖代码解释器、AI 代理、日志采集或代码分析产物
- 下游影响用户审查流程、问题定位流程和结果归档流程

### 当前风险或未决问题

- 当前仍未确定后续是否扩展到多类过程源，而不只服务于 Jigsaw 调试回放
- 当前仍未确定后续是否需要多人共享审查标记
- 如果 Jigsaw 调试产物字段继续演化，导入器和展示契约需要同步维护

## Jigsaw 求解器回放首版

### 影响的模块、接口、配置、阶段

- 新增实现仓库顶层独立子工程 `code_process_viewer/`
- 新增导入接口 `POST /api/import/jigsaw-debug`
- 新增会话查询、图查询、节点详情、表格、审查、报告导出接口
- 新增 SQLite 模型：
  - `sessions`
  - `nodes`
  - `edges`
  - `table_snapshots`
  - `source_refs`
  - `review_checkpoints`
  - `reviews`
  - `imports`
- 新增运行期目录：
  - `runtime/db/`
  - `runtime/cache/`
  - `runtime/tmp/`
- 首版只影响 Jigsaw 调试回放，不介入 `city_jigsaw_solve` 求解执行链路

### 影响的测试入口或验收口径

- 验收入口从泛化“过程图”收敛为真实 `jigsaw_solver_debug/<runId>` 导入与回放
- 需要额外验证：
  - `trace.json` 缺失时报错
  - 预览图缺失时节点可看但会提示证据不完整
  - 审查标记刷新后仍保留

### 影响的上下游系统

- 上游固定依赖：
  - `country_designer_mcp` 的 `city_jigsaw_solve` 请求链路
  - Java 侧 Jigsaw 调试导出产物
- 下游固定影响：
  - Jigsaw 问题定位流程
  - 人工审查结论沉淀
  - 调试回放报告归档

### 当前风险或未决问题

- 真实调试目录样本路径不固定，首版需要用户手工提供 `artifact_path`
- 若同一 `debug_run_id` 被重复覆盖导入，需要明确是否保留既有审查结论
- 当前首版未纳入实时刷新与会话对比，复杂调试场景仍需要人工多开比对
