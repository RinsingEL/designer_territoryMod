# TerraSense 测试入口

## 第一阶段验收

| 验收项 | 标准 |
| --- | --- |
| MC 自动样本产出 | 一条命令或自动测试入口能稳定扫描三类固定样本，并生成 `scan_manifest.json` |
| 三类结构覆盖 | 样本覆盖 `single`、`template`、`single_template` |
| 结构发现 | 能扫描原版和至少一个结构 mod 的 `.nbt` structure |
| 批量截图 | 选择一批结构后能连续生成截图和 `data.json`，固定样本至少输出 `angle_01.png` 与 `angle_02.png` |
| 扫描配置 | 每个样本输出 `scan_config.json`，记录样本类型、摄影场和截图参数 |
| 断点续扫 | 中断后可从 `scan_index.json` 继续 |
| AI 初标 | 可批量生成 AI 建议，失败不阻塞人工审核 |
| 人工审核 | 可脱离 MC 查看截图、修改标签并写入 `review.json` |
| Studio 启动 | TerraSense Studio 可独立启动并读取 `ScanWorkspace` |
| Studio 图片识别 | Studio 可在 `mock` 模式下稳定生成 `ai_suggestion.json`，并支持真实 vision API 模式 |
| 导出 catalog | 能生成 StructureBinder 当前可读的 `C3_5_StructureCatalog.preprocessed.json` |
| C7 查询验证 | `StructureTemplateQueryService` 能按功能召回人工审核结构 |
| C8 jigsaw 验证 | 至少一个带 jigsaw 的样本能被 C8/Jigsaw 链路读取 connector 真值 |

## 功能版本验收

| 功能包 | 验收边界 |
| --- | --- |
| v1 MC 自动样本产出 | 固定三类样本均有 `data.json`、`scan_config.json`、两张截图和 manifest 索引；`template/single_template` 样本包含完整 runtime jigsaw 字段 |
| v2 Studio 工作台 | 不启动 MC 也能打开 workspace、浏览截图和硬事实、保存 `review.json` |
| v3 Studio 图片识别初标 | Studio 可通过 mock 和真实 vision API 两种路径生成 `ai_suggestion.json`，人工确认后写入 `review.json` |

## 建议测试样本

- 原版村庄住宅。
- 原版工作站建筑。
- 带 jigsaw 的道路或房屋结构。
- 一个结构 mod 的住宅类结构。
- 一个结构 mod 的地标或大型建筑。
- 一个应被拒绝的低质量或不适合城市生成的结构。
