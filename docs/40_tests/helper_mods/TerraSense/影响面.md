# TerraSense 影响面

## 结构策展工具

- 影响模块
  - TerraSense 结构扫描、自动样本产出、截图、AI 初标、人工审核、导出。
  - TerraSense Studio 本地工作台。
  - StructureBinder C3.5 catalog。
  - StructureBinder C7-C9 结构消费链路。
- 影响接口与产物
  - `ScanWorkspace`
  - `scan_manifest.json`
  - `scan_index.json`
  - `scan_config.json`
  - `data.json`
  - `ai_suggestion.json`
  - `review.json`
  - `StructureCurationExport`
  - `C3_5_StructureCatalog.preprocessed.json`
  - `StructureProfile`
- 上下游影响
  - 上游依赖结构 mod/datapack 和 MC runtime template 数据。
  - 下游影响城市功能区结构候选、C7 工头计划、C8 jigsaw 求解和 C9 落地。
- 当前风险
  - 名字关键词标记质量不足，不能作为高质量城市生成真值。
  - AI 初标可能幻觉，必须人工审核。
  - jigsaw orientation 不能压扁成 facing，否则会污染 C8 求解。
  - 若导出格式与 StructureBinder catalog 不兼容，标注无法闭环。
  - 客户端截图链路受渲染、窗口和文件写入时序影响，必须保留 mock/硬事实测试入口作为稳定验收兜底。

## MC 自动测试环境

- 影响模块
  - TerraSense Forge 客户端命令或自动测试入口。
  - 结构放置、摄影场清理、相机控制和截图输出。
  - runtime jigsaw 读取和 connector 派生逻辑。
- 影响接口与产物
  - `scan_manifest.json`
  - `structures/<safe_structure_id>/data.json`
  - `structures/<safe_structure_id>/scan_config.json`
  - `structures/<safe_structure_id>/screenshots/angle_01.png`
  - `structures/<safe_structure_id>/screenshots/angle_02.png`
- 影响测试入口
  - v1 验收必须固定三类样本：`single`、`template`、`single_template`。
  - 自动测试至少检查文件存在、manifest 状态、jigsaw 字段完整性。
- 当前风险
  - 自动进入存档和截图渲染可能受本机 MC 客户端状态影响。
  - 三类样本的结构 id 需要选取稳定存在于当前测试环境的结构。

## TerraSense Studio

- 影响模块
  - Studio 本地 Web 前端。
  - Studio 本地文件 API。
  - 动态术语表、AI 初标、人工审核和 review 写入。
- 影响接口与产物
  - `vocabulary.json`
  - `ai_suggestion.json`
  - `review.json`
  - `StructureCurationExport`
- 影响测试入口
  - v2 验收要求不启动 MC 也能读取 workspace、浏览截图和硬事实、保存 `review.json`。
  - v3 验收要求 mock AI 与真实 vision API 两条路径都能写出 `ai_suggestion.json`。
- 当前风险
  - AI 初标只能作为建议，不能直接覆盖人工审核真值。
  - 本地文件 API 必须限制 workspace 根目录，避免误写无关文件。
