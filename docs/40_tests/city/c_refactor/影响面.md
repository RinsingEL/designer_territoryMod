# C 大重构影响面（2026-05-06 版）

## C1 城市画布

- 影响模块
  - C1 宏观底图生成。
  - step=8 预览图导出、图例、坐标映射。
  - 后续 C2 对城市边界和功能区多边形的消费。
- 影响接口与产物
  - 新增或替代 `UrbanIntentMap.json`。
  - 新增 image2 输入底图、原始输出图、CV overlay 预览。
- 上下游影响
  - 上游依赖真实地形、领土范围、水系和高度。
  - 下游影响 C2 权重场、C3 ownership、C6 area 安全规范。
- 当前风险
  - image2 颜色漂移导致 CV 解析不稳定。
  - 图像坐标与世界坐标映射若不稳定，会污染后续所有阶段。
  - C1 不应承担施工级禁区判断，过度检查会让宏观规划变重。

## C2 多边形功能区权重表

- 影响模块
  - C2 语义输出。
  - C3 ownership / C4 road / C5 area 的输入模型。
- 影响接口与产物
  - 新增 `district_polygons`。
  - 新增 `district_function_weights`。
  - 新增 `placement_context_hints`。
- 上下游影响
  - 上游消费 C1 意图图。
  - 下游影响 C6 area 安全规范和 C7 结构搜索。
- 当前风险
  - 若权重场缺少解释来源，后续调试会很难判断为什么某地生成某类结构。
  - 第一版不应把权重场复杂化成连续栅格热力图，优先保持多边形可读性。
  - 权重不是硬约束，需要避免下游误用。

## C3 意图图到 Ownership 转换

- 影响模块
  - C3 ownership / block seed 生成。
  - C4 road sketch adapter 的道路提示输入。
  - C5 area 图的片区来源。
  - C6 area 安全规范的功能区和权重来源。
- 影响接口与产物
  - 新增或扩展 `c3_ownership_blocks.json`。
  - 新增 `conversion_report`，记录裁剪、清理、合并、丢弃和 warning。
  - 新增程序重渲染的 `intent_overlay_preview.png`。
- 上下游影响
  - 上游消费 `UrbanIntentMap` 和 C2 `district_function_weights`。
  - 下游向 C4-C6 提供世界坐标下的 district / ownership / block seed。
- 当前风险
  - 半透明图、渐变、阴影和底图混色不能作为机器解析源。
  - 坐标映射错误会导致所有后续 ownership 和 block 偏移。
  - 多边形清理过度可能改变功能区语义，必须记录到 `conversion_report`。
  - 功能权重需要继承到 block，不应被压扁成单标签。

## C4 道路草图识别与网络化

- 影响模块
  - C4 road sketch adapter。
  - C5 area 图规划。
  - C6 area 安全规范的临路、背街、滨水和广场边上下文。
  - C7 工头计划中的入口、主路、桥头、广场边结构选择。
- 影响接口与产物
  - 新增 `C4RoadNetwork.json`。
  - 新增 `road_nodes`、`road_edges`、`road_corridors`、`intersection_areas`。
  - 新增 `frontage_hints` 和 `solve_report`。
  - 新增道路 overlay 预览。
- 上下游影响
  - 上游消费 C3 `road_hints / anchor_refs / block_seeds`。
  - 下游向 C5 提供 road corridor 和 blocked area，向 C6/C7 提供 frontage 语义。
- 当前风险
  - 当前实现中的旧 `C4_FunctionPlan.json` 是语义计划，不是道路网络；实现时需要避免命名和数据真值混淆。
  - C4 若自行重规划道路，会绕开 image2 的城市设计意图。
  - 道路过密会把功能区切碎，导致 C5 `buildable_area` 质量下降。
  - 道路过稀会让港口、广场、市场等关键锚点不可达。
  - 普通道路不应穿水；跨水必须显式建模为桥或滨水连接。
  - C4 不应提前选择桥、城门、道路装饰等具体结构模板。

## C5 Area 图规划与保留策略

- 影响模块
  - C5 area planner。
  - C6 area 安全规范的候选空间来源。
  - C7 工头计划中的地标、广场、庭院、港口 yard、碎片空间使用。
- 影响接口与产物
  - 新增 `C5AreaPlan.json`。
  - 新增 `district_area / buildable_area / reserved_area / residual_area`。
  - 新增 `layout_profile / split_policy / size_band / reserved_strategy`。
  - 新增 area overlay 预览和 `planning_report`。
- 上下游影响
  - 上游消费 C3 `ownership / block seed / function_weights` 和 C4 `road_corridors / frontage_hints / intersection_areas`。
  - 下游向 C6 提供安全规范的候选空间，向 C7 提供 area 级布局意图。
- 当前风险
  - 如果 C5 切得过碎，后续结构会塞不下，C6/C7 会进入低质量补救。
  - 如果 C5 切得过粗，会浪费 image2 的道路和功能区设计。
  - `reserved_area` 如果缺失，广场、桥头、港口 yard、城门口和地标庭院容易被普通小建筑吃掉。
  - `residual_area` 如果伪装成普通建设地，会导致 C7 反复选择无法落地的结构。
  - C5 不应反向修改 C4 道路真值，也不应提前选择最终结构模板。

## C6 Area 安全规范

- 影响模块
  - C6 area safety generator。
  - 旧 `CityC6Stages` 中可复用的 footprint 统计、connector 预留、growth buffer 和地形检查能力。
  - C7 工头计划、C8 placement / jigsaw、C9 执行复核的安全边界输入。
- 影响接口与产物
  - 新增 `C6AreaSafetySpec.json`。
  - 新增 `AreaSafetySpec / HardConstraint / SoftRisk / ExceptionPolicy / ReviewRequirement`。
  - 旧 `C6_BuildAreaLayout.json`、`RectGuidance`、`PrimaryModule` 只作为迁移兼容或 debug/fallback。
- 上下游影响
  - 上游消费 `C5AreaPlan`、`C4RoadNetwork`、step=1 局部地形和 `StructureProfile`。
  - 下游向 C7/C8/C9 提供统一安全规范；C7 仍可直接读取 step=1 地图。
- 当前风险
  - C6 如果重新输出矩形槽位主方案，会重新压死 C5/image2 的设计自由度。
  - C6 如果写建筑建议，会和 C7 工头计划职责重复。
  - C6 如果阻止 C7 读取 step=1 地图，会变成错误的信息闸门。
  - C6 安全规范不等于落地保证，最终仍由 C8/C9 真实 placement、jigsaw 和 runtime 校验决定。
  - 硬约束越界必须人工评估或由 C8/C9 给出合法特例证据，不能被 C6 或 C7 悄悄绕过。

## LLM 复查与策略配置

- 影响模块
  - C1/C2 意图复查。
  - C4 道路网络复查。
  - C5 area 策略配置。
  - 人工评估与 AI 自修边界。
- 影响接口与产物
  - 可新增 `intent_review / road_network_review / area_strategy_review`。
  - 可在 `C5AreaPlan.llm_review` 中记录 area 策略建议。
  - 需要保存 input refs、accepted/rejected suggestions、warning 和人工评估标记。
- 上下游影响
  - 上游需要提供原图、overlay 和结构化 JSON 摘要。
  - 下游只能消费被接受的建议，不能把 LLM 原始文本当施工真值。
- 当前风险
  - LLM 如果只看图不看 JSON，容易把视觉判断误当坐标真值。
  - C2 阶段若提前配置最终道路，会和 C4 的道路识别职责冲突。
  - C5 阶段若允许 LLM 修改道路网络，会破坏 C4RoadNetwork 真值边界。
  - 当视觉表现不满意、修复绕路或多轮自修不收敛时，需要人工评估。

## 结构标记辅助 Mod

- 影响模块
  - 开局前结构扫描、结构 catalog、结构查询、C7 候选召回、C8 详情消费。
- 影响接口与产物
  - `StructureProfile`
  - `structure_pool_search`
  - `structure_template_detail`
  - 低置信度标注复核队列
- 上下游影响
  - 上游来自开局前扫描、NBT、AI 标注、人工覆盖。
  - 下游影响 C7/C8 的候选质量和 AI 上下文大小。
- 当前风险
  - AI 标注权重可能漂移，需要 `confidence / evidence / review_state`。
  - 硬事实和语义权重必须分开，否则会污染施工合法性。
  - 结构画像不应被放进 C7-C9 的逐节点循环里反复重标。

## C7-C9 主链

- 影响模块
  - 单功能区 step=1 精细底图。
  - C6 area 安全规范。
  - C7 工头计划和结构选择。
  - C8 placement / jigsaw 求解。
  - C9 执行和失败回写。
- 影响接口与产物
  - C6 需要在具体功能区 step=1 后生成 area 安全规范。
  - C7 需要从单标签查询升级为权重召回。
  - C8 需要保留结构详情按需展开。
  - C9 可把失败区域回写为后续 negative mask。
- 当前风险
  - 一次性改完整链路风险高，建议先做 C1.5 兼容适配。
  - 若不保留旧链路兼容，历史调试和回归会断档。
