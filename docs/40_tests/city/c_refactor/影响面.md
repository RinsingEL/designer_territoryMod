# C 大重构影响面（2026-05-06 版）

## C1 城市画布

- 影响模块
  - C1 宏观底图生成。
  - step=8 预览图导出、图例、坐标映射。
  - 后续 C2 对城市边界和功能区多边形的消费。
- 影响接口与产物
  - 新增或替代 `UrbanIntentMap.json`。
  - 新增 image2 输入底图、原始输出图、CV overlay 预览。
- 上下游影响
  - 上游依赖真实地形、领土范围、水系和高度。
  - 下游影响 C2 权重场、C3 ownership、C6 建造区。
- 当前风险
  - image2 颜色漂移导致 CV 解析不稳定。
  - 图像坐标与世界坐标映射若不稳定，会污染后续所有阶段。
  - C1 不应承担施工级禁区判断，过度检查会让宏观规划变重。

## C2 多边形功能区权重表

- 影响模块
  - C2 语义输出。
  - C3 ownership / C4 road / C5 parcel 的输入模型。
- 影响接口与产物
  - 新增 `district_polygons`。
  - 新增 `district_function_weights`。
  - 新增 `placement_context_hints`。
- 上下游影响
  - 上游消费 C1 意图图。
  - 下游影响 C6 footprint budget 和 C7 结构搜索。
- 当前风险
  - 若权重场缺少解释来源，后续调试会很难判断为什么某地生成某类结构。
  - 第一版不应把权重场复杂化成连续栅格热力图，优先保持多边形可读性。
  - 权重不是硬约束，需要避免下游误用。

## C3 意图图到 Ownership 转换

- 影响模块
  - C3 ownership / block seed 生成。
  - C4 road solver 的道路提示输入。
  - C5 parcel / area 的片区来源。
  - C6 footprint budget 的功能区和权重来源。
- 影响接口与产物
  - 新增或扩展 `c3_ownership_blocks.json`。
  - 新增 `conversion_report`，记录裁剪、清理、合并、丢弃和 warning。
  - 新增程序重渲染的 `intent_overlay_preview.png`。
- 上下游影响
  - 上游消费 `UrbanIntentMap` 和 C2 `district_function_weights`。
  - 下游向 C4-C6 提供世界坐标下的 district / ownership / block seed。
- 当前风险
  - 半透明图、渐变、阴影和底图混色不能作为机器解析源。
  - 坐标映射错误会导致所有后续 ownership 和 block 偏移。
  - 多边形清理过度可能改变功能区语义，必须记录到 `conversion_report`。
  - 功能权重需要继承到 block，不应被压扁成单标签。

## 结构标记辅助 Mod

- 影响模块
  - 开局前结构扫描、结构 catalog、结构查询、C7 候选召回、C8 详情消费。
- 影响接口与产物
  - `StructureProfile`
  - `structure_pool_search`
  - `structure_template_detail`
  - 低置信度标注复核队列
- 上下游影响
  - 上游来自开局前扫描、NBT、AI 标注、人工覆盖。
  - 下游影响 C7/C8 的候选质量和 AI 上下文大小。
- 当前风险
  - AI 标注权重可能漂移，需要 `confidence / evidence / review_state`。
  - 硬事实和语义权重必须分开，否则会污染施工合法性。
  - 结构画像不应被放进 C7-C9 的逐节点循环里反复重标。

## C6-C9 主链

- 影响模块
  - 单功能区 step=1 精细底图。
  - C6 footprint budget。
  - C7 工头计划和结构选择。
  - C8 placement / jigsaw 求解。
  - C9 执行和失败回写。
- 影响接口与产物
  - C6 需要在具体功能区 step=1 后从结构画像统计 footprint。
  - C7 需要从单标签查询升级为权重召回。
  - C8 需要保留结构详情按需展开。
  - C9 可把失败区域回写为后续 negative mask。
- 当前风险
  - 一次性改完整链路风险高，建议先做 C1.5 兼容适配。
  - 若不保留旧链路兼容，历史调试和回归会断档。

## 建议优先级

1. C1.5 ImageIntent Adapter。
2. `UrbanIntentMap` 契约和 overlay 预览。
3. 开局前结构画像标注和 `structure_pool_search` 原型。
4. C2 多边形功能区权重表。
5. C3 意图图到 ownership / block 兼容转换。
6. C6/C7 消费权重场和结构画像。
7. C8/C9 接入失败回写和更完整的闭环。
