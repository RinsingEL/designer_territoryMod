## 第一阶段验收入口

| 阶段 | 验收内容 |
| --- | --- |
| C1.5 ImageIntent Adapter | 同一 `step = 8` 宏观底图可生成 `UrbanIntentMap.json` 和 overlay 预览 |
| CV 解析 | 固定 palette 图可稳定解析城市边界、功能区多边形、道路草图、锚点 |
| 基础检查 | 越领土边界、覆盖其他城市、碎片多边形、断路等问题可输出 warning 或 repair action |
| C3 兼容转换 | 新 `UrbanIntentMap` 和 C2 权重表可临时转成现有 C3/C6 可消费输入 |
| C4 道路草图网络化 | 给定 image2 road sketch、C3 road hints 和 anchor refs，可生成连通、不过界、可交给 C5 的 `C4RoadNetwork` |
| C5 area 图规划 | 给定 `C3OwnershipBlocks` 和 `C4RoadNetwork`，可生成不过碎、可追踪、带保留策略的 `C5AreaPlan` |
| C6 area 安全规范 | 给定 `C5AreaPlan`、step=1 局部地图、道路上下文和结构画像，可生成 `C6AreaSafetySpec`，且不输出设计方案 |
| LLM 复查与配置 | 给定原图、overlay 和 JSON 摘要，可输出分阶段 review，不越级修改道路、area 或施工真值 |

## 建议测试类型

| 类型 | 说明 |
| --- | --- |
| golden image 测试 | 固定输入图，断言解析出的图层数量、面积和关键锚点 |
| 契约测试 | `UrbanIntentMap`、`C3OwnershipBlocks` 和 `StructureProfile` 字段兼容性 |
| mask 稳定性测试 | 固定纯色 `intent_mask.png` 多次解析得到一致 district / road / anchor 结果 |
| C3 转换测试 | 给定 `UrbanIntentMap` 和 C2 权重表，断言 ownership / block seed 坐标、ID 和权重继承正确 |
| C4 道路图测试 | 给定 `C3OwnershipBlocks` 和 road sketch，断言 road graph 连通性、entry 覆盖、锚点可达和 road corridor 输出 |
| C4 边界测试 | 断言普通道路不越城市边界、不穿水；跨水道路必须标记为 `bridge` 或 `waterfront` |
| C4 回退测试 | 当道路主结构缺失或严重断裂时，断言输出 `requires_redraw`，而不是自行重规划主路 |
| C5 area 规划测试 | 给定 C3/C4 产物，断言 `district_area / buildable_area / reserved_area / residual_area` 分类、父子关系和面积统计正确 |
| C5 保留策略测试 | 断言广场、桥头、港口 yard、城门口、地标庭院不会被普通 `buildable_area` 吃掉 |
| C5 碎片处理测试 | 断言极小碎片标为 `residual_area`，不会计入普通建设容量 |
| C6 安全规范契约测试 | 断言每个 area 输出 `hard_constraints / soft_risks / exception_policy / review_requirements` |
| C6 不做设计测试 | 断言 `C6AreaSafetySpec` 不包含建筑建议、最终模板、坐标、旋转、施工顺序或工头计划 |
| C6 旧矩形兼容测试 | 断言旧 rect guidance 只能作为 debug/fallback，不成为新 C6 主真值 |
| LLM review 契约测试 | 断言 review 只能输出建议、warning、人工评估标记，不直接覆盖结构化几何真值 |
| 搜索排序测试 | 给定功能区权重表，断言结构搜索 top N 符合预期 |
| 回归测试 | 同一城市旧 C1 与新 C1.5 的 C6-C9 成功率和失败原因对比 |
| 人工验收 | overlay 预览是否能看出城市外形更自然、功能混合更合理 |
