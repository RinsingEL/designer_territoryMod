# 主模块建造测试入口

## 当前测试目标

主模块系统当前重点验证三件事：

1. `C7 -> C8` 的计划与会话主链是否稳定。
2. `C8` 节点提交、失败、重试是否可持续推进。
3. `city_jigsaw_solve` 是否能输出可读证据，并在需要时正确衔接执行层。

## 功能级测试清单

| 编号 | 功能 | 测试项 | 预期结果 |
| --- | --- | --- | --- |
| M1 | C7 工头计划 | 调用 `city_c7_generate` 生成 group 计划 | 成功生成 `c7_selection.json`，含 `foreman_plan / phase_list / start_node` |
| M2 | C7 工头计划 | group 模式校验失败 | 返回 `422` 或 `409`，并回包校验信息 |
| M3 | C8 会话生成 | 调用 `city_c8_generate` | 成功生成 `c8_foundation.json`，返回 `session_state / active_node / queue_summary` |
| M4 | C8 数据读取 | 调用 `city_c8_data` | 能读到最新 group 级会话 |
| M5 | C8 节点提交 | 对 `active_node` 发起 `city_c8_submit` | 成功写入 `validated_nodes`，并回包 `submit_result / queue_summary` |
| M6 | C8 失败重试 | 对失败节点调用 `city_c8_retry` | 节点回到可继续推进状态，`active_node` 同步更新 |
| M7 | Jigsaw 求解 | 调用 `city_jigsaw_solve` | 生成 `trace.json` 与步骤预览，返回结构化 `solve_result` |
| M8 | Jigsaw 立即落地 | `city_jigsaw_solve` 开启 `apply_now` | 返回 `apply_result`，并能在执行层产物中看到结果同步 |

## 当前证据入口

| 类型 | 位置 |
| --- | --- |
| `C7` 产物 | `group/<groupId>/c7_selection.json` |
| `C8` 产物 | `group/<groupId>/c8_foundation.json` |
| `Jigsaw` 调试产物 | `group/<groupId>/jigsaw_solver_debug/<runId>/` |
| 下游执行结果 | `group/<groupId>/c9_placement.json` 与 `C9_BuildQueue` |

## 建议联调顺序

1. 先跑 `city_c7_generate`
2. 再跑 `city_c8_generate`
3. 对 `active_node` 反复执行 `city_c8_submit / city_c8_retry`
4. 需要 child 结构时再进入 `city_jigsaw_solve`
5. 涉及立即落地时，再回看 `C9` 执行层系统文档
