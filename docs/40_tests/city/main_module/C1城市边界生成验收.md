# C1 城市边界生成验收

## 验收目标

验证新的 C1 城市边界生成方案是否能替代旧 4 方向 chunk 泛洪扩张作为城市设计画布，并为后续 C2/C3/C8 主链提供更自然、可校验、非监督且适配不同国度设定的边界输入。

## 输入

- 城市中心点
- `targetChunkCount`
- 国境/允许范围
- 水体、海岸、坡度、roughness
- 已有城市占用信息
- 开局高层配置：style / density / waterfront preference / fallback policy

## 必验项

| 编号 | 项目 | 验收方式 | 通过标准 |
| --- | --- | --- | --- |
| B1 | 边界自然度 | 查看 `C1_boundary_preview.png` 与方正度指标 | 外轮廓不再呈明显 4 方向 chunk 膨胀形态 |
| B2 | 设定敏感地形处理 | 对比水体、海岸、坡地与国度设定 | 水城/港口/山城等设定可利用对应地形，普通陆城则倾向避让高风险地形 |
| B3 | 层级健康 | 检查 heat layers / validation | core、urban、transition 不应无解释地明显竖条化；buffer 可吸收水体、坡地、边缘风险 |
| B4 | 非监督审查 | 构造异常输入 | 系统能自动 pass / retry / auto_fix / fallback，不要求人工确认 |
| B5 | fallback | 强制制造不可用中心点 | 能生成保守 fail_safe 城市画布或明确降级，不中断世界生成 |
| B6 | 兼容性 | 读取旧链路输入 | 能继续产出 claimed chunks 兼容字段，旧 C2/C3/C5 可 fallback |
| B7 | C3 衔接 | 使用新 boundary plan 生成 C2/C3 输入 | C3 AI 草图不再被旧方框边界锁死 |

## 阻塞条件

出现以下情况应视为失败：

- 边界大面积越出国境且未被 auto_fix
- 与已有城市或同批次城市发生硬重叠且未重排
- 中心点不属于当前国度且未重新选址
- 边界为空、不可解析或碎成无法连通的孤岛
- 连续失败后没有 fallback / fail_safe

以下情况不应默认阻塞，而应按国度设定输出 warning / risk tag：

- core 或 urban 靠近水体
- 边界包含陡坡、森林、沼泽、悬崖等特殊地形
- transition / preserve 偏小但城市设定为高密度或硬边界
- 外轮廓较方正但国度设定允许规则化城市
