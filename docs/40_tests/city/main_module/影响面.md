# 主模块建造影响面

## C7 工头计划

- 影响模块
  - `CityController.handleCityC7Generate / handleCityC7Data`
  - `CityC7Stages`
  - `CityC7Validation`
- 影响接口与产物
  - `city_c7_generate`
  - `city_c7_data`
  - `C7_TemplateSelection.json`
  - `group/<groupId>/c7_selection.json`
- 上下游影响
  - 上游依赖 `C6Layout`
  - 下游直接影响 `C8` 会话生成
- 当前风险
  - `strict_tag_source` 过滤过严时，仍可能把问题前移到候选模板不足

## C8 节点会话与提交

- 影响模块
  - `CityController.handleCityC8Generate / Data / Submit / Retry`
  - `CityC8Stages`
- 影响接口与产物
  - `city_c8_generate`
  - `city_c8_data`
  - `city_c8_submit`
  - `city_c8_retry`
  - `C8_FoundationPlan.json`
  - `group/<groupId>/c8_foundation.json`
- 上下游影响
  - 上游依赖 `C7` 工头计划与 `C6` geometry
  - 下游直接影响 `C9` 可消费节点与 placement
- 当前风险
  - 失败重试与历史 `validated_nodes` 并存时，仍需要持续关注节点去重与审计可读性

## C8 子节点 Jigsaw 求解

- 影响模块
  - `CityController.handleCityJigsawSolve`
  - `CityVanillaJigsawAdapterService`
  - `CityJigsawSolverDebugTrace`
  - `CityJigsawSolverPreviewExporter`
- 影响接口与产物
  - `city_jigsaw_solve`
  - `group/<groupId>/jigsaw_solver_debug/<runId>/`
  - 可选 `apply_now` 下游执行结果
- 上下游影响
  - 上游依赖最新 `C8` placement、catalog 与 runtime 模板
  - 下游会把 solved placement 交给执行层系统
- 当前风险
  - 当前仍只支持单步、水平 `jigsaw`
  - catalog 缺失 child 模板元数据时会直接阻塞求解
