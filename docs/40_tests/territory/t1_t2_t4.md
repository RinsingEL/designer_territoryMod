# T1/T2/T4 国度步骤验收

## 必验项

| 编号 | 项目 | 验收方式 | 通过标准 |
| --- | --- | --- | --- |
| T1-1 | W4 前置门禁 | 清空或缺失 region cache 时运行 T1 | 返回或记录 W4 detail cache 缺失，不生成假候选 |
| T1-2 | 候选生成 | 有效 W4 后调用 `/t1_candidates_for_continent` | 每个目标国度返回 `workflow_state / blocked / cluster_selected`，并可读候选 bundle |
| T1-3 | 簇选择固化 | 调用 `/t1_select_cluster` | `T1_Status.json` 中 `clusterSelected=true`，旧 T2 产物被清理 |
| T2-1 | T1 前置门禁 | 未选择 T1 簇时调用 T2 | T2 blocked，提示先完成 T1 |
| T2-2 | 方向候选 | 调用 `/t2_direction_candidates` | 写出 `T2_DirectionCandidates.json`，响应包含方向候选或 blocked 原因 |
| T2-3 | 方向选择固化 | 调用 `/t2_select_direction` | `T2_Status.json` 中 `pointSelected=true`，territory config 与 T2 capital summary 已写出 |
| T4-1 | T3 前置门禁 | 未完成 T3 时运行 T4 | 返回或记录 `T4 requires territory results from T3` |
| T4-1a | 当前实例主权恢复 | 重启 client 后调用 `/territory_status` | 已写入 canonical T3 的当前实例 `total_area > 0`，无需再次手动导入 |
| T4-2 | 地形扫描产物 | 完成 T3 后运行 T4 | 每个有效国度写出 T4 summary/dat 与 batch report |
| T4-3 | C1 窗口读取 | 调用 `/territory/t4_window` | 成功返回 `terrain / biome_composition_window / samples`；缺 dat 时返回 `404` |
| T4-3a | 默认中心吸附 | 在 capital 落出 T4 覆盖区时调用 `/territory/t4_window` 且不传 `center_x/z` | 返回 `200`，`window.reanchored=true`，并给出最近 T4 采样点窗口而不是空窗口 |
| T4-4 | 历史 T3 导入 | 调用 `/territory/t3_import` | `dry_run` 返回摘要；真实导入后 `runtime_restored=true` |

## 自动验证

在实现仓库运行：

```powershell
.\gradlew.bat test --tests "*Territory*"
.\gradlew.bat test --tests "*WorkflowControllerTest"
.\gradlew.bat compileJava compileTestJava
```

`TerritoryWorkflowStagesTest`、`TerritoryStageArtifactsTest`、`TerritoryControllerTest`、`WorkflowControllerTest` 与 `TerritoryResultRepositoryTest` 负责覆盖门禁、响应构造、T4 参数兼容、T3 dat 读写/导入与零面积保护。T4 真实地形扫描仍需要 Minecraft server 与 T3 结果做手动回归。

## 手动回归入口

```powershell
Invoke-RestMethod -Method Get -Uri http://127.0.0.1:5000/t1_blueprint
Invoke-RestMethod -Method Post -Uri http://127.0.0.1:5000/t1_candidates_for_continent -Body '{"continent_id":1}' -ContentType 'application/json'
Invoke-RestMethod -Method Post -Uri http://127.0.0.1:5000/t1_select_cluster -Body '{"territory_id":"han","continent_id":1,"cluster_id":1}' -ContentType 'application/json'
Invoke-RestMethod -Method Post -Uri http://127.0.0.1:5000/t2_direction_candidates -Body '{"territory_id":"han","continent_id":1}' -ContentType 'application/json'
Invoke-RestMethod -Method Post -Uri http://127.0.0.1:5000/t2_select_direction -Body '{"territory_id":"han","continent_id":1,"direction":"center"}' -ContentType 'application/json'
Invoke-RestMethod -Method Post -Uri http://127.0.0.1:5000/t3_run_continent -Body '{"continent_id":1}' -ContentType 'application/json'
Invoke-RestMethod -Method Post -Uri http://127.0.0.1:5000/territory/t3_import -Body '{"source_territory_id":"han","target_territory_id":"han@c1","dry_run":true}' -ContentType 'application/json'
Invoke-RestMethod -Method Get -Uri http://127.0.0.1:5000/territory_status
Invoke-RestMethod -Method Post -Uri http://127.0.0.1:5000/workflow/run -Body '{"stage_id":"T4","sample_stride":2,"max_chunks_per_territory":-1,"loaded_only":false}' -ContentType 'application/json'
Invoke-RestMethod -Method Post -Uri http://127.0.0.1:5000/territory/t4_window -Body '{"territory_id":"han@c1"}' -ContentType 'application/json'
```

示例里的 `territory_id / continent_id / cluster_id / direction` 必须替换为当前候选响应中的真实值。若目标实例来自历史基础 id 恢复路径，应先完成一次真实 `t3_import`，再重启 client，用 `/territory_status` 验证自动恢复。
