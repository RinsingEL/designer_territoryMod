# W3/W4 世界扫描验收

## 必验项

| 编号 | 项目 | 验收方式 | 通过标准 |
| --- | --- | --- | --- |
| W3-1 | 全图扫描前置 | 未运行 world scan 时调用 W3 | 返回或记录 `W3 requires global scan data`，不得写出假产物 |
| W3-2 | 大陆/海洋聚类 | 有 `lastScanData` 后调用 `/world/w3/cluster` 或 `run_workflow_stage(W3)` | 写出 `ContinentMeta.json` 与 `OceanMeta.json`，内存中有 `lastClusterMap` |
| W4-1 | region cache 前置 | 未运行 region scan 时调用 W4 export | 返回或记录 `W4 requires region cache` |
| W4-2 | 地形摘要导出 | 对有效 region 调用 `/world/w4/region_scan` 后再调用 `/world/w4/export` | 写出 `TerrainSummary.json` 与 `TerrainFacts.dat` |
| W4-3 | T1 前置可用 | W4 后调用 `get_world_atlas` 与 T1 候选入口 | atlas/summary 可读，T1 不再因 W4 detail cache 缺失阻塞 |

## 自动验证

当前 W3/W4 主流程依赖 Minecraft server、世界扫描缓存和 region cache，不能只靠纯 JVM 单元测试完整覆盖。提交前至少运行编译检查：

```powershell
.\gradlew.bat compileJava compileTestJava
```

## 手动回归入口

```powershell
Invoke-RestMethod -Method Post -Uri http://127.0.0.1:5000/world/scan/start -Body '{"radius_chunks":128}' -ContentType 'application/json'
Invoke-RestMethod -Method Post -Uri http://127.0.0.1:5000/world/w3/cluster -Body '{}' -ContentType 'application/json'
Invoke-RestMethod -Method Post -Uri http://127.0.0.1:5000/world/w4/region_scan -Body '{"region_id":1}' -ContentType 'application/json'
Invoke-RestMethod -Method Post -Uri http://127.0.0.1:5000/world/w4/export -Body '{"region_id":1}' -ContentType 'application/json'
```

`region_id` 应使用 W3 输出中的有效大陆 id。若本机没有启动游戏内 HTTP server，`127.0.0.1:5000` 会连接失败，此时只能做编译和纯单元测试。
