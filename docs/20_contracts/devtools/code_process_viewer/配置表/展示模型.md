# 展示模型

## 目标

本文件定义代码过程浏览可视工具首版对前端暴露的核心展示模型。

当前首版具体落地对象为 `Jigsaw` 求解器调试回放，因此字段命名优先贴近 `trace.json.debug_trace`。

原则：

- 前端只接收后端整理好的展示数据。
- 前端不自己推导节点依赖、状态含义和表格结构。
- 所有“可审查对象”都尽量结构化。

## Session

| 字段 | 类型 | 必填 | 说明 |
| --- | --- | --- | --- |
| `session_id` | string | 是 | 一次分析会话唯一标识 |
| `title` | string | 是 | 会话标题 |
| `project_name` | string | 是 | 项目名 |
| `branch` | string | 否 | 所在分支 |
| `commit_id` | string | 否 | 对应提交号 |
| `debug_run_id` | string | 是 | 对应一次 `jigsaw_solver_debug/<runId>` |
| `artifact_path` | string | 是 | 调试目录绝对路径 |
| `status` | enum | 是 | `success / warning / failed` |
| `created_at` | datetime | 是 | 创建时间 |
| `updated_at` | datetime | 是 | 更新时间 |
| `entry_node_ids` | string[] | 否 | 流程入口节点 |
| `tags` | string[] | 否 | 场景标签 |

## 节点生成约束

| 项目 | 约束 |
| --- | --- |
| 节点来源 | 后端按 `trace.json.debug_trace` 自动生成 |
| 节点粒度 | 一个 `step` 对应一个节点 |
| 首版连边 | 相邻步骤固定生成 `sequence` 边 |
| AI 负责内容 | 仅补充可读摘要，不发明节点事实 |
| AI 不负责内容 | 节点事实、关联文件、状态结果 |

## 首版接口输出建议

| 接口 | 说明 |
| --- | --- |
| `GET /api/sessions` | 会话列表 |
| `GET /api/sessions/{id}` | 单个会话概览 |
| `GET /api/sessions/{id}/graph` | 图节点和边 |
| `GET /api/nodes/{id}` | 节点详情 |
| `GET /api/nodes/{id}/tables` | 节点表格快照 |
| `POST /api/nodes/{id}/review` | 提交人工审查标记 |
| `GET /api/sessions/{id}/report` | 导出报告 |
| `POST /api/import/jigsaw-debug` | 导入一次 `jigsaw_solver_debug/<runId>` |

## 当前约束

1. 前端不自己拼装流程图业务语义。
2. 前端不自己推导字段含义。
3. 首版前端只展示后端已经归一化的 Jigsaw 调试数据。
