# W3/W4 世界扫描契约

## 阶段边界

| 阶段 | 目的 | 前置 | 主要产物 |
| --- | --- | --- | --- |
| W3 | 基于全图扫描缓存做大陆/海洋聚类 | `ScanResultHolder.lastScanData` 已存在 | `world/W3/ContinentMeta.json`、`world/W3/OceanMeta.json`、内存中的 `lastClusters / lastOceanRegions / lastClusterMap` |
| W4 | 基于局部 region cache 导出地形摘要、预览图与压缩地形事实 | W3 已完成，且 `regionCacheMap` 非空 | `world/W4/TerrainSummary.json`、`world/W4/TerrainFacts.dat`、W4 预览图索引 |

W3/W4 属于世界事实准备层，不创建国度、不分配主权，也不直接进入 C1 城市阶段。

## HTTP 入口

| 入口 | 方法 | 作用 |
| --- | --- | --- |
| `/world/scan/start` | `POST` | 启动世界全图扫描，生成 W3 所需的全局扫描缓存 |
| `/world/w3/cluster` | `POST` | 基于当前全图扫描缓存执行 W3 聚类并导出大陆/海洋元数据 |
| `/world/w4/region_scan` | `POST` | 按 region 执行 W4 局部 detail scan |
| `/world/w4/region_status` | `POST` | 读取指定 region 的 W4 detail cache 状态 |
| `/world/w4/export` | `POST` | 基于当前缓存导出 W4 产物 |
| `/world_atlas` | `GET` | 读取 W4 世界图集与摘要 |
| `/world_summary` | `GET` | 读取 W4 世界摘要 |
| `/terrain_summary` | `GET` | 读取 W4 地形摘要 |

## W3 产物

`ContinentMeta.json` 与 `OceanMeta.json` 的数组项包含：

```json
{
  "id": 1,
  "type": "CONTINENT",
  "metrics": {
    "area_pixels": 12000,
    "avg_height": 74,
    "max_relief": 31,
    "roughness": "1.42",
    "center_x": 320,
    "center_z": -128
  },
  "climate": {
    "avg_temp": "0.68",
    "temp_range": "[0.20, 0.95]",
    "temp_distribution_4x4": []
  },
  "ecology": {
    "dominant_biomes": []
  }
}
```

## W4 产物

`TerrainSummary.json` 至少包含：

```json
{
  "regions": [
    {
      "id": 1,
      "minX": -2048,
      "minZ": -2048,
      "w": 256,
      "h": 256,
      "step": 16
    }
  ],
  "preview_images": {}
}
```

`TerrainFacts.dat` 为压缩 NBT，按 region 保存 `id / minX / minZ / w / h / step`，以及可选的 `slope / roughness / tpi` 矩阵。

## 下游约束

- T1 依赖 W4，缺少 W4 detail cache 时不得继续生成候选。
- T3 失败提示中的 W3/W4 scan cache 指的就是 W3 `lastClusterMap` 与 W4 region detail cache。
- C1 需要已有国度上下文，不能直接用 W3/W4 结果跳过 T1/T2/T3/T4。
