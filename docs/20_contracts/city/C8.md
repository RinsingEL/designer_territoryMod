# C8 Contract

## 当前定位

`C8` 负责维护结构施工会话，并承担 child 节点的 connector / jigsaw 求解与落地前程序校验。

## 当前稳定输出重点

- `foreman_plan`
- `session_state`
- `active_node`
- `node_queue`
- `validated_nodes`
- `failed_attempts`
- `foundation_context`

## 当前稳定输入重点

- 来自 `C7` 的工头计划
- 当前节点决策：
  - `node_id`
  - `selected_template_id`
  - `selected_connector_dir`
  - `selected_rotation`
  - `terrain_relax_profile`
- 仅对以下场景保留直接坐标输入：
  - `start` 节点
  - 显式 debug / override

## 当前边界

- `C8` 负责节点决策会话、connector / jigsaw 求解与程序校验
- `C8` 不再承担整图复杂排列枚举
- `C8` 的输出需要被 `C9` 消费成可执行建造计划
- child 节点应在 `C8` 阶段完成 `origin` / `rotation` / bounds 求解，而不是把手工 `x / z` 长期当主路径
- 下列拒绝应优先在 `C8` 出现，而不是延后到 `C9`：
  - `out_of_area`
  - 真实 bounds 冲突
  - 地形不合法
  - 不满足收尾条件

## 当前兼容策略

- `start` 节点保留直接 `place` 路径
- child 节点默认走求解器路径
- 手工 `x / z` 只作为调试或显式覆盖手段，不再视为 child 主契约
