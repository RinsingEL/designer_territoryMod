# C9 Contract

## 当前定位

`C9` 负责把已校验且已求解完成的结构节点转成可执行建造任务，并进入运行时执行链。

## 主链

1. 从 `C8.validated_nodes` 取可执行节点
2. 生成 `BuildTask`
3. 写入 `BuildQueue`
4. 等区块满足执行条件后落地

## 当前关键输出

- `C9_Placement`
- `C9_Decoration`
- `C9_BuildQueue`
- `queue_summary`

## 当前运行时职责

- 父节点依赖检查
- chunk readiness
- 真实模板 bounds 冲突检查
- 地形预处理与清空
- 结构实际落地
- 按 `placement.origin_offset.y` 解释默认落点高度
- 落地后 jigsaw `final_state -> air fallback` 收口
- retry / blocked

## 当前边界

- `C9` 不负责决定“下一个 child 用哪个结构”
- `C9` 不负责重新随机消费 jigsaw pool
- `C9` 只消费 `C8` 已经求解完成的 placement
- `start` 节点直接放置与 child 节点求解放置允许并存
- 如果世界状态在 `C8` 之后发生变化，`C9` 仍保留 runtime 最终拒绝权

## 当前高度与清理口径

- `placement.origin_offset.y = 0`：默认贴表。
- `placement.origin_offset.y = -1`：默认埋入 1 格。
- `city_group_surface_clear` 默认下限为 `foundation.base_y - 1`；只要显式传入 `min_y / max_y`，就严格按请求值执行。
- `BuildTerrainPreparationService` 当前在清理当前节点 bounds 时，会跳过同一 `BuildQueue` 中已落地父节点（`DONE / BUILDING`）的保护 bounds，避免后续 child 落地时把父节点主体误挖成“半截结构”。

## 后续扩展入口
