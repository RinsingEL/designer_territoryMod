## 当前坐标口径

- `placement.origin_offset.y = 0` 表示结构贴表放置。
- `placement.origin_offset.y = -1` 表示结构默认下沉 1 格。
- 未显式标注的模板默认按 `0` 处理，不再由运行时猜测“普通房屋统一埋 1 格”。
- start 节点若存在 `chosen_start_connector_id` 且 `runtime_fallback_used != true`，则优先按运行时 start connector 真值定高：`start_origin_y = foundation.base_y - chosen_start_connector.local_y`。
- 读取 `group/<groupId>/c8_foundation.json` 时，会自动归一化这类 start 的 `y` 与 `placement.y`；当前存档无需重生 `C8` 即可修正 stale start 高度。

## 当前会话收口规则

- `validated_nodes` 与 `placements` 当前按 `node_id` 保持唯一；重复提交同一 start/节点时，会覆盖旧记录而不是继续累加同名节点。
- `city_jigsaw_solve` 成功后，真实 `vjigsaw_*` child 会立即写回当前 group 的 `c8_foundation.json`，并继续展开下一层 `READY_FOR_AI` 节点。
- solved child / grandchild 的 `template_pool_id` 与下一层 `candidate_template_ids` 当前优先使用 runtime pool 真值；若与 catalog 冲突，不阻断求解，但会在 `node_debug` 与 `solve_result` 中显式留下 mismatch 记录。

## 当前优先级规则

- `city_c8_generate` 优先读取 `group/<groupId>/c7_selection.json`，找不到时才回退城市级 `C7`。
- `city_c8_submit`、`city_c8_retry` 与 `city_jigsaw_solve` 优先读取 `group/<groupId>/c8_foundation.json`，找不到时才回退城市级 `C8`。
- `city_jigsaw_solve` 求解成功后，会把真实 child 写回当前 `C8` 会话树，并同步更新 `queue_summary / active_node / validated_nodes / failed_attempts`。
- `city_c8_submit(apply_now=true)` 当前会额外写出专用 `trace.json`，其中包含 `04_world_snapshot`，用于比对同一 runtime bounds 的落地前后世界块统计，专门排查 start/普通节点“只落半截”问题。
