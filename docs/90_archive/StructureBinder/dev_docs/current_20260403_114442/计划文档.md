# 本次任务标题

C9 施工执行层拆分与落地流水线方案初稿

## 最终计划摘要

围绕当前 `C9` 的结构落地链路，整理一份新的执行层方案初稿，重点收口“建造队列管理”“落地方法封存”“地形预处理/原世界清理”“区块加载门控”等职责边界，为后续拆分类与次级建筑复用做准备。

## 实施步骤

1. 回收当前 `C9` 主链中已落地的队列、执行器、放置入口与高度口径。
2. 结合最新讨论，将施工执行层重构目标整理成独立方案。
3. 明确建议拆分的服务层、各自职责，以及与现有类的映射关系。
4. 将本轮计划、需求、修改记录与方案正文写入当前任务目录。

## 验收标准

- 已形成一份独立的执行层方案初稿。
- 已明确“队列管理 / 区块门控 / 地形预处理 / 结构落地 / 运行时校验”的拆分方向。
- 已明确当前主链哪些属于稳定基线，哪些属于后续应拆出的逻辑。
- 已对 `getHeight` / 高度口径与区块预存-延后落地模式给出可执行建议。

## 关键假设

- 当前主链仍继续沿用 `BuildQueue -> Executor -> StructureInjector`。
- 本轮先做方案整理，不直接开始执行层代码重构。
- 后续次级建筑也应尽量复用这套执行层，而不是另起一套落地器。
