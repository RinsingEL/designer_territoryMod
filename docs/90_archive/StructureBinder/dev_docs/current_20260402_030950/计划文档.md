# 当前 C8 大修改验收计划

## 本次任务标题

验收当前 C8 大修改，并优先使用仓库内可落地的 MCP 测试链路进行验证。

## 最终计划摘要

围绕当前工作区中与 C8/C9、MCP 接口和施工队列相关的改动开展一次验收检查，先确认改动边界与预期，再执行可运行的 MCP/单元测试与必要的静态核对，最后给出是否通过、未覆盖风险和后续建议。

## 实施步骤

1. 检查当前工作区状态、已有开发文档与 C8 相关改动范围。
2. 定位仓库内 MCP 服务、测试脚本、单元测试和验收入口。
3. 执行 MCP 相关测试与补充测试，记录结果和失败信息。
4. 回读计划、需求、修改记录三份文档，形成验收结论。

## 验收标准

- 能明确说明当前 C8 大修改覆盖了哪些模块、阶段和接口。
- 至少完成一轮与 MCP 相关的实际验证，并补充必要的自动化测试结果。
- 能指出阻塞验收的问题、回归风险和未覆盖范围。
- 最终结论与证据链可回溯到本轮文档和测试结果。

## 关键假设

- 用户所说的“当前 C8 大修改”对应当前工作区里尚未提交的 C8/C9、MCP 和施工流程改动。
- 若全局 MCP 资源为空，则优先使用仓库中的本地 MCP 实现、脚本或测试入口完成验证。
- 本轮以验收和测试为主，不主动改动与验收无关的业务实现。
