# C6-C9 Polygon-First 去 BBox 化计划

## 计划摘要
把当前 `bbox + mask_bbox + buildableGroups + rect` 的多层几何体系继续收敛成更清晰的 `polygon-first` 方案，不只让 C6 停止把 bbox 当主语义，也要让 C8/C9 的求解、复核、预览与对外接口一起摆脱 bbox 依赖。

## 实施步骤
1. 删除 `mask_bbox` 作为决策输入主约束。
2. 弱化 `bbox` 为内部辅助信息，不再作为 AI 主设计边界。
3. 保留 footprint guidance，但语义改成结构尺寸建议。
4. 将 C6 校验改成 `polygon + 最小覆盖率`。
5. 将 C8 改成基于 area block / polygon 集合求解，不再用 bbox 做主轴和越界判断。
6. 将 C9 改成基于 area block / polygon 集合复核与入队，不再继承 bbox 越界语义。
7. 同步更新 C8/C9 预览、对外字段和工作流文档。

## 验收标准
- 决策输入不再以 `mask_bbox` 为核心。
- C6 校验不再在此阶段裁决最终结构建造性。
- C8 不再以 bbox 作为主轴判断和 placement 越界硬约束。
- C9 不再把超出 polygon area 的 placement 入队或落地。
- C8/C9 预览与文档不再保留 bbox / mask 语义。

## 关键假设
- 保留 bbox 作为内部性能辅助是允许的。
- 本次不重写 C7，只改 C8/C9 对 C6 产物的消费方式。
- `C6` 数据模型里的 `bbox` 字段可以暂时保留，但在 C8/C9 中应成为死字段。
