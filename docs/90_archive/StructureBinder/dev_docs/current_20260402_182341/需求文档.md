# 用户本次需求摘要

用户希望在新分支上优先推进次级建筑，而不是道路；并要求先基于仓库里过去已经写过的案子，整理一份次级建筑独立方案初稿供后续评审。

## 用户特别强调的约束

- 使用中文对话。
- 文件默认按 UTF-8 打开和保存。
- 回复中的文件链接使用 markdown 绝对路径，格式固定为 `/c:/...`。
- 先看最近 git 和既有文档，不要重复发明已经写过的口径。
- 次级建筑希望单独成案。
- 当前更倾向先做次级建筑，因为道路计划外包。

## 不做的事情

- 本轮不直接开始次级建筑代码实现。
- 本轮不把道路系统写成主开发目标。
- 本轮不把次级建筑重新并回主体建筑那套复杂排列求解路线。

## 当前决定的工作边界

- 只整理次级建筑独立方案初稿。
- 尽量复用 `v0.1.5`、`v0.1.6` 以来已经出现的结论。
- 重点收口“次级可建造块 -> `single` 落地”的最小闭环。
