# 本次任务标题

次级建筑 `single` 化落地方案初稿整理

## 最终计划摘要

基于仓库内已有关于“主体建筑与次级建筑分层”“次级建筑默认输出为可建造块”“次级建筑 `single` 化落地”等既有结论，整理一份新的次级建筑独立方案初稿，作为新分支后续开发的起点。

## 实施步骤

1. 回收现有提交与文档中已经拍板的次级建筑口径。
2. 结合当前 `C6/C8/C9` 已落地链路，整理次级建筑最小闭环方案。
3. 输出一份独立初稿，明确目标、阶段边界、数据流、最低规则和待确认问题。
4. 将本轮计划、需求与修改记录同步写入当前任务目录。

## 验收标准

- 已形成一份独立的次级建筑方案初稿。
- 已明确次级建筑与主体建筑、道路系统的边界。
- 已明确“次级可建造块 -> `single` 落地”的主线。
- 已列出后续真正需要继续拍板的问题，而不是重复旧结论。

## 关键假设

- 当前优先做次级建筑，不优先做道路。
- 道路后续更偏向外包，因此本案只保留与道路对接有关的最小接口意识。
- 次级建筑第一阶段默认走简化路径，不复用主体建筑那套复杂工头式 AI 规划。
