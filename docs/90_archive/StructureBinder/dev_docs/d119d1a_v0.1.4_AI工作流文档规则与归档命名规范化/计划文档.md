# AGENTS 工作流文档约束计划

## 计划摘要
把现有 `dev_docs` 升级成统一的 AI 工作流文档机制，并把规则写进 `AGENTS.md`。以后每次从计划模式进入开工，都必须先落一组当前任务文档；开发过程中持续追加修改记录；验收时必须回读这些文档；提交后再把当前目录归档成 `git短提交号_版本号_中文标题` 的格式。

## 实施步骤
1. 在 `AGENTS.md` 中补充 AI 工作流文档规则。
2. 明确 `dev_docs/current_YYYYMMDD_HHMMSS` 的临时目录规范。
3. 明确三份固定文件：
   - `计划文档.md`
   - `需求文档.md`
   - `修改记录.md`
4. 明确验收前必须回读这三份文档。
5. 明确提交成功后目录必须重命名为 `git短提交号_vX.Y.Z_中文标题`。
6. 立刻在当前任务中创建一套真实的 `dev_docs/current_...` 文档作为工作流样板。

## 验收标准
- `AGENTS.md` 已写入完整工作流约束。
- 当前任务已生成 `dev_docs/current_20260329_181630/` 三份文档。
- 文档内容与本次任务目标一致。
- 后续 agent 可以直接按此规则执行。

## 关键假设
- 继续使用 `dev_docs` 作为根目录。
- 临时目录统一用 `current_YYYYMMDD_HHMMSS`。
- 一个提交对应一个归档目录。
