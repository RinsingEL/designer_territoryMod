# 本次需求文档

## 用户本次需求摘要
延续上一线程，继续用问答方式把 `C8` 从“排列方式/抽象求解器”路线收口为“LLM规划 + 程序校验 + LLM修补”的工作流。用户本轮希望先不写复杂 schema，而是直接起草一份初版方案，清楚说明工头计划、单起点施工、节点队列、失败回包和 AI 重试边界。

## 用户特别强调的约束
- 总规阶段更像“工头计划/草图计划”，只做指导作用，不直接给完整施工结果。
- 当前只保留一个 `start`，避免提前处理并行问题。
- `phase` 要保留，用于指导 subagent 按施工顺序推进。
- 节点是什么结构必须明确，而且由上一步链路规定，不由当前 subagent 现猜。
- 节点可用模板来自当前拼图方块自带模板池，不开放跨池乱选。
- 错误回包必须详细，失败后由 AI 决定换模板、换方向或终止。
- AI 唯一允许放宽的是结构自身的地形落地限制，其他硬约束不放宽。

## 不做的事情
- 本轮不继续扩展 `arrangement_type`、`layout_solver`、`core/wing/terminal/junction` 之类的中间语义层。
- 本轮不设计多 `start`、多节点并发施工、复杂阶段图。
- 本轮不直接修改 `C7/C8/C9` 代码实现。

## 当前决定的工作边界
- 方案重点放在“工头计划 -> 单节点施工 -> 程序校验 -> AI重试 -> 子节点入队”的闭环。
- 先收最小必需字段和流程，不追求一步到位设计完整 schema。
- 文档落点先放在当前任务目录，作为后续更新正式工作流文档与 prompt 的依据。
