# C8切换为工头计划与逐节点施工工作流初版

## 最终计划摘要
本轮不继续扩展 `arrangement_type` 或抽象求解器层，而是把 `C8 -> C9` 工作流改写成“工头计划 + 单起点 + 节点施工队列 + 程序校验 + AI重试”的初版方案文档。目标是先把最小可运行流程写清楚，让后续文档、prompt 和接口都围绕这个流程收口。

## 实施步骤
1. 回收本轮问答里已经拍板的口径，整理出最小工作流主线。
2. 在当前任务目录起草一份初版方案，明确总规阶段、节点施工阶段、失败重试与队列流转。
3. 将本轮已确认的节点输入边界、失败处理规则和程序职责同步写入修改记录。
4. 回读计划文档、需求文档、修改记录与方案草案，确认内容一致且边界清楚。

## 验收标准
- 文档中明确写出“工头计划只做指导，不直接给完整施工结果”。
- 文档中明确写出“当前只支持单一 `start` 节点 + 队列推进”。
- 文档中明确写出节点施工输入最小集合、失败回包要求和可放宽范围。
- 文档中明确写出程序与 AI 的职责边界，不再混入旧的排列枚举扩展思路。

## 关键假设
- 本轮主要产出方案文档，不同步修改 `C7/C8/C9` 运行时代码。
- 当前版本先不处理多 `start`、多工区并行、节点并发施工等问题。
- 节点结构目标由阶段计划与父节点拼图方块模板池共同约束，AI只在合法范围内选方案。
