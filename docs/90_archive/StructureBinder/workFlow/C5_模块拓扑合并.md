# C5 模块拓扑合并

## 当前状态

- 状态：已实现
- 入口：`CityController.handleCityC5Generate(...)`

## C5 在当前项目里的真实分层

当前 C5 是一个非常“程序主导”的阶段：

1. 读取 C4 结果
2. 按拓扑与功能规则自动合并
3. 输出 group、merge log 和预览图

AI 不直接给 group，只能通过上游 C4 或 merge policy 间接影响结果。

## 三、程序具体怎么做

### 1. 先读取 C4 plan

优先读取：

- `C4_FunctionPlan.validated.json`

如果没有，再退回原始：

- `C4_FunctionPlan.json`

### 2. 解析 merge policy

`parseC5MergePolicy(...)` 支持：

- `cross_layer_merge`
- `adjacency_mode`
- `min_district_area_chunks`
- `allow_cross_function_absorb_for_tiny`
- `split_disconnected_group`
- `min_compactness`

### 3. 生成初始 draft

`generateC5(...)` 会先基于：

- district primary function
- adjacency
- 是否允许跨 layer

构造基础 group draft。

### 4. 吸收碎片 district

`absorbTinyDistricts(...)` 会处理面积过小的 district：

- 优先并入最合适的大组
- 在策略允许时可跨功能吸附
- 并把原因写入 `merge_log`

### 5. 拆分不连通 group

如果启用 `split_disconnected_group`：

- 程序会检查 group 内是否存在多个离散 component
- 如果有，就拆成多个 group

### 6. 输出质量统计

最终还会计算：

- `fragment_reduction_ratio`
- `disconnected_groups`
- `low_compactness_groups`

## 四、输入

- `city_id`
- `C4 plan`
- `merge policy`
- 可选 C2 scan / C3 ownership（仅预览使用）

## 五、输出

- `C5_ModuleGroups.json`
- `C5_MergeLog.json`
- C5 module preview

输出字段重点：

- `group_id`
- `function`
- `layer`
- `district_ids`
- `district_numeric_ids`
- `area_blocks`
- `centroid`
- `connectivity.component_count`
- `connectivity.compactness`

## 六、AI 决策和程序计算的边界

### 1. AI 决策的部分

- 当前不直接给 group
- 最多通过：
  - 上游 C4 功能输出
  - merge policy
  影响结果

### 2. 程序计算的部分

- group draft 构造
- tiny absorb
- disconnected split
- compactness / quality 统计
- merge log 生成
