# C6 可建造矩形生成

## 当前状态

- 状态：已实现
- 当前语义已经切到“AI 决策主矩形 + 程序校验闭环”
- 入口：
  - `handleCityC6Generate(...)`
  - `handleCityC6RectPrepare(...)`
  - `handleCityC6RectSubmit(...)`

## C6 在当前项目里的真实分层

当前 C6 明确分成两段：

1. 程序生成 polygon-first build area、默认 layout 骨架和 AI 决策输入
2. AI 或人工提交矩形，程序做严格校验并决定是否写入最终 layout

### 程序具体怎么做

#### 1. 准备输入

程序会读取：

- `C5 groups`
- `Stage1 height data`
- 可选 `C2 scan`
- `buildableGroups`（仅 fallback 参考）
- 可选已有的 layout/candidates/validation

#### 2. 重新生成 polygon-first build area

`CityC6Stages.generate(...)` 会：

- 优先基于 C5 group 与 C3 ownership 结果生成 polygon-first build area
- 当 ownership 数据缺失时，才回退参考 C1 的 buildableGroups
- 对每块可建区域计算：
  - `polygon_reference_bbox`（仅参考）
  - `centroid`
  - `avg_height`
  - `ascii_map`
- 给每块区域生成：
  - `build_area_id`
  - `build_area_numeric_id`

#### 3. 默认只为每个 group 选最佳 build area

程序会记录 `bestAreaByGroup`：

- 如果同组有多块 build area
- 默认只把面积最大的那块作为主消费对象

#### 4. 生成默认 layout 骨架

`buildPlans(...)` 会给每个 group 生成一个空的 `LayoutPlan`：

- 默认 `fill_style`
- 默认 `PlazaRingParams`
- 默认 `rect_sizes`
- 但此时还没有真正被接受的 `primary_modules`

#### 5. 生成 AI 决策输入

`buildDecisionInputs(...)` 会输出每组的：

- `group_id`
- `build_area_id`
- `polygon_area_blocks`
- `polygon_reference_bbox`
- `centroid`
- `rect_limit`
- `min_total_primary_area_ratio`

当前关键常量：

- `RECT_ATTEMPT_LIMIT = 3`
- `MIN_TOTAL_PRIMARY_AREA_RATIO = 0.50`
- `MIN_COVERAGE_RATIO = 0.80`

## 二、Rect Submit：AI 提交矩形并校验

### 入口

| 类 | 方法 | 说明 |
| --- | --- | --- |
| `server.mcp.CityController` | `handleCityC6RectSubmit(HttpExchange)` | 提交矩形并做验证 |

### 核心类与关键方法

| 类 | 方法 | 作用 |
| --- | --- | --- |
| `world.city.stage.c6.CityC6Validation` | `validateSubmission(...)` | 校验当前提交 |
| `world.city.stage.c6.CityC6Validation` | `validateRect(...)` | 校验单个矩形 |
| `world.city.stage.c6.CityC6Validation` | `normalizeRect(...)` | 规范化矩形 |
| `world.city.stage.c6.CityC6Stages` | `applyAcceptedDecision(...)` | 把矩形写成最终 primary_modules |
| `world.city.stage.c6.CityC6Stages` | `upsertCandidate(...)` | 写候选历史 |
| `world.city.stage.c6.CityC6Stages` | `upsertValidation(...)` | 写校验历史 |

### 程序具体怎么校验

#### 1. 先校验 attempt 序号

- `attempt_index` 必须顺序递增
- 超过 `RECT_ATTEMPT_LIMIT` 会终止

#### 2. 支持三种 decision_mode

- `submit_rects`
- `keep_current`
- `no_primary_module`

#### 3. submit_rects 的校验逻辑

对每个矩形：

- `normalizeRect(...)` 先补全 `min/max/cx/cz/w/h`
- 再遍历矩形覆盖到的每个 block
- 统计：
  - `inside_functional_blocks`
  - `total_rect_blocks`
- 得到：
  - `coverage_ratio = inside / total`

单矩形通过条件：

- `w > 0`
- `h > 0`
- `coverage_ratio >= 0.80`

整组提交通过条件：

- 全部矩形都合法
- `total_primary_area_ratio > 0.50`

结构适配信息仍会生成，但不再作为 C6 接受与否的硬判定；最终结构可建造性由后续 C7/C8/C9 决定。

#### 4. 通过后如何写回 layout

`applyAcceptedDecision(...)` 会把每个 `RectDecision` 转成 `PrimaryModule`：

- `module_id`
- `rect_id`
- `anchor`
- `w/h`
- `min/max`
- `template_hint`

并把 `layout.validated = true`。

## 五、AI 决策和程序计算的边界

### 1. AI 决策的部分

- 给定当前 group，决定是否：
  - 放 1~N 个主矩形
  - 保留当前方案
  - 不放主模块
- 决定每个矩形：
  - 中心点
  - 尺寸
  - 空间角色

### 2. 程序计算的部分

- 生成 polygon-first build area
- 生成默认 layout 骨架
- 输出预览图
- coverage 校验
- 面积占比校验
- 尝试次数与状态机控制
- 写回 validated layout
