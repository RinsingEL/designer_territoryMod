# C9 结构落地与装饰计划

## 当前状态

- 状态：已实现基础版本
- 当前语义已经基本切到“严格按 C8 placements 落地”
- 入口：`CityController.handleCityC9Generate(...)`

## C9 的当前真实边界

当前 C9 已经不再负责“重新决定怎么放”：

- 它只消费 `C8 plan`
- 决定 dry-run 还是实际改世界
- 负责运行时校验、结构落地和失败回退

## 三、程序具体怎么做

### 1. 先建立 area -> foundation 和 area -> block set 映射

`generate(...)` 会把：

- `C8Plan.foundations`
- `C6 index`

整理成：

- `foundationByArea`
- `blocksByArea`

同时会先复核每个 placement footprint 是否完整落在对应的 area block 集合内；不满足时直接标记 `runtime_out_of_area`，不会进入入队/落地链路。

### 2. 对每个 build area 调 placeArea(...)

`placeArea(...)` 的输入是：

- `areaBlocks`
- `foundation`
- `budget`
- `resultItem`

如果 `foundation.placements` 不为空，就会走“按 placements 落结构”的主路径。

### 3. placements 如何执行

执行前会：

- 按 `build_order` 排序
- 建立 `byId`
- 建立 `placedStates`
- 建立 `terminatedBranchRoots`

对每个节点：

- 若该节点属于已终止分支，直接跳过并记录原因
- 若不是 dry-run，则先做 `validateRuntimePlacement(...)`
- 校验通过后才调用 `StructureInjector.spawnStructureAtBlock(...)`

### 4. 运行时会校验什么

`validateRuntimePlacement(...)` 当前会检查：

- footprint 是否仍在 area block 集合内
- footprint 是否和已放节点冲突
- parent 是否已经 terminalized
- connector 是否匹配

常见 reject reason 包括：

- `runtime_footprint_collision`
- `runtime_out_of_area`
- `parent_already_terminalized`
- `runtime_connector_mismatch`

### 5. 失败后如何回退

如果某个节点放置失败：

- 会尝试 `tryTerminalizeAncestor(...)`
- 恢复父节点 snapshot
- 用 `fallback_terminal_template_id` 替换父节点
- 把这一支 branch 标记为 terminalized

所以 C9 当前已经具备“失败后裁断分支”的运行时兜底能力。

### 6. decoration plan 如何生成

`inferDecorStrategy(...)` 当前是规则映射：

- `TERRACE -> SLOPE_GARDEN`
- `PLATFORM_WITH_RETAINING_WALL -> WALL_TORCHES`
- `PLATFORM -> PLAZA_LANTERNS`
- 其他 -> `NATURAL_SCATTER`

这部分目前还是简化版，不是完整装饰系统。

## 四、输入

- `city_id`
- 可选 `group_id`
- `C6 summary/index`
- `C8 plan`
- `apply_blocks`
- `max_blocks`

## 五、输出

- `C9_Placement.json`
- `C9_DecorationPlan.json`
- group 级：
  - `c9_placement.json`
  - `c9_decoration.json`

输出字段重点：

- `processed_areas`
- `changed_blocks_total`
- `planned_nodes`
- `placed_structures`
- 每个节点的：
  - `placed`
  - `reason`

## 六、AI 决策和程序计算的边界

### 1. AI 决策的部分

- 当前基本没有
- 上游 AI 的影响已经被固化到：
  - C7 arrangement
  - C8 placements

### 2. 程序计算的部分

- dry-run / 实际落地
- 运行时碰撞和连接校验
- 结构放置
- 失败回退与 branch terminalize
- 简化 decoration 策略生成

### 3. 当前代码和理想语义的差距

- 理想上 C9 就是纯粹执行 C8 plan。
- 当前已经大体符合，但仍带有一些 fallback 和 decoration 简化逻辑。
- 装饰部分仍然很轻，主价值目前还是结构主模块落地。
