# C3 多边形区划种子与分区

## 当前状态

- 状态：已实现基础版本
- 入口：`CityController.handleCityC3Generate(...)`
- 但和 `solution.md` 里“先全域 polygon，再按 layer 二次拆分”的 C3 还不完全等价

## C3 在当前项目里的真实分层

当前代码里，C3 更接近这两段：

1. `Stage2Processor` 把 district buildable 统计转成意图摘要
2. `CityC3OwnershipIO` 把城市 block 映射到最近 district 的 owner 栅格

所以当前 C3 更像“ownership raster + polygon preview”，还不是完整的 polygon 分层系统。

## 三、程序具体怎么做

### 1. Stage2Processor 先生成 district intent

`CityStage2Processor.compute(...)` 会：

- 汇总全城市的：
  - `globalAvgHeight`
  - `globalAvgRoughness`
  - `globalMinHeight`
  - `globalMaxHeight`
- 对每个 district 生成 `DistrictIntent`

关键推导逻辑：

- `step = max(2, round((maxHeight - minHeight) / 6.0))`
- `baseLevelFromDensity(...)` 先推导 `HIGH/MID/LOW`
- `adjustLevel(...)` 再根据 `buildablePercent` 和 `avgRoughness` 调整
- `targetHeight = avgHeight + offset`
- `transition = chooseTransition(...)`

当前 transition 规则：

- `buildablePercent < 0.25` -> `CLAMP`
- `roughness >= 2.5` -> `TERRACE`
- 否则 -> `RAMP`

### 2. CityC3OwnershipIO 生成 owner 栅格

`CityC3OwnershipIO.compute(city)` 会：

- 先计算城市整体 bounds
- 创建 `owner[width][height]`
- 遍历所有 claimed chunk 展开的 16x16 block
- 对每个 block 调 `nearestDistrictId(worldX, worldZ, districts)`
- 把它归给最近的 district id

所以当前 owner 逻辑本质上是“最近 district center 归属”，不是严格 polygon clipping。

### 3. 导出 polygon preview

`CityC3PolygonPreviewExporter.export(...)` 会把：

- 城市扫描图
- ownership raster
- district 信息

叠加成预览图，供后续步骤复核。

## 五、输出

- `polygon_preview`
- `intent_count`

## 六、AI 决策和程序计算的边界

### 1. AI 决策的部分

- 当前实现里基本没有
- 如果按方案，AI 最多只能调策略，不直接手工点种子

### 2. 程序计算的部分

- district intent 推导
- 最近 district 归属
- owner raster 生成
- polygon preview 输出

## 七、当前代码和方案的差异

- `solution.md` 的 C3 是：
  - 先全域 polygon
  - 再按 layer 二次拆分
- 当前代码里还没有显式的 `global_polygon_id -> layered_polygon` 数据结构
- 现在更像：
  - 先有 district
  - 再生成 block owner 栅格和预览图
