阅读代码时一定先看 `RuntimeOptions.legacyTerrainScan`。

## 三、新语义：城市 bootstrap

### 程序具体怎么做

`runCapitalCityBootstrap(...)` 会：

1. 遍历 `TerritoryManager.getAllResults()`
2. 对每个 territory 调 `buildCityBootstrapConfig(...)`
3. 调 `CityManager.get().createCity(cityCfg)`
4. 生成 `mode=CITY_BOOTSTRAP` 的摘要
5. 写 T4 summary 和 batch report

### buildCityBootstrapConfig(...) 的真实字段映射

- `territoryId = territory.id`
- `continentId = territory.regionId`
- `centerX/centerZ = capitalX/capitalZ`
- `allowWaterCity = options.cityAllowWater`
- `targetChunkCount = options.cityTargetChunks > 0 ? value : -1`
- `bias = parseBias(options.cityBias, BALANCED)`
- `ecology = parseEcology(options.cityEcology, ADAPTIVE)`
- `density = options.cityDensity`，默认回落到 `medium`

所以当前默认 T4 并不让 AI 再决定建城点，而是直接沿用领地首都。

### T3 自动触发时的默认参数

`WorkflowController` 在 T3 成功后会把 T4 改成：

- `sampleStride = 4`
- `maxChunksPerTerritory = 256`
- `loadedOnly = true`
- `legacyTerrainScan = false`
- `bootstrapCity = true`

这说明默认工作流已经把 T4 当成“城市接力阶段”。

## 四、旧语义：block 级战略扫描

### 程序具体怎么做

`runLegacyTerrainScan(...)` 的流程：

1. 汇总每个 territory 的 `claimedChunks + wildChunks`
2. `estimateTotalBlocksFromChunks(...)` 估算总采样量
3. 用 `ProgressTracker` 做心跳和 25% 进度广播
4. 对每个 territory 调 `analyzeTerritory(...)`
5. 导出 summary + dat

### analyzeTerritory(...) 计算了什么

#### 1. 采样列数据

每个 territory 会生成一批 block column。

每列采样：

- `height`
- `temperature`
- `biomeId`

如果 `loadedOnly=false`：

- 走 `ChunkGenerator + RandomState`
- 不强制加载 chunk

如果 `loadedOnly=true`：

- 走已加载 chunk + `Heightmap`

#### 2. 计算边境场和首都场

- `computeBorderDistances(...)`
  - 以边境格为 0 做 BFS
- `computeCapitalDistances(...)`
  - 从首都最近格开始做 BFS

#### 3. 计算局部坡度

`localSlope(...)` 用四邻格高差平均值估算。

#### 4. 计算战略值

`strategicValue(...)` 当前公式：

- `score = 0.6 * borderNorm + 0.4 * (1.0 - capitalNorm)`
- 再压缩到 `0~255`

含义：

- 越远离边境越安全
- 越接近首都越有中心性

#### 5. 输出哪些摘要

summary 里会形成：

- `border_security`
- `hinterland_depth`
- `max_distance_to_border`
- `max_distance_to_capital`
- `choke_points`
- `biome_composition`

DAT 里会保存：

- `x/z`
- `height`
- `slope`
- `temperature`
- `distBorder`
- `distCapital`
- `strategic`

## 七、AI 决策和程序计算的边界

### 1. AI / 调用方能影响的部分

- 是否走 `legacyTerrainScan`
- `sampleStride`
- `maxChunksPerTerritory`
- `loadedOnly`
- `bootstrapCity`
- `cityTargetChunks`
- `cityBias`
- `cityDensity`
- `cityEcology`
- `cityAllowWater`

### 2. 程序计算的部分

- 自动触发 T4
- 构造 `CityConfig`
- 创建城市
- 或执行战略扫描
- 计算战略值、距边境距离、距首都距离

### 3. 当前代码和方案的差异

- `solution.md` 中 T4 更偏战略场。
- 当前默认工作流中，T4 更偏“首都城市 bootstrap”。
- 旧扫描能力还在，但已经不是默认路径。
