# C8 排列坐标求解与基台计划

## 当前状态

- 状态：已实现基础版本，仍在重构联调
- 入口：`CityController.handleCityC8Generate(...)`

## C8 的当前真实边界

- `C7` 决定“怎么排”
- `C8` 负责“把排法解成具体坐标和基台策略”
- 当前最核心的程序逻辑集中在：
  - `CityC8Stages.generate(...)`
  - `CityC8ArrangementEngine.solve(...)`

| 类 | 方法 | 说明 |

## 三、程序具体怎么做

### 1. 先建立索引

`generate(...)` 会先把输入整理成：

- `blocksByArea`
- `arrangements` 索引
- `selections` 索引
- `plansByArea`
- `area geometry`

这样每个 build area 都能同时拿到：

- `LayoutPlan`
- `GroupArrangementDecision`
- `TemplateSelectionItem`

### 2. 调用 ArrangementEngine 求 placements

对每个可消费的 build area：

- 找到对应 `layoutPlan`
- 调用 `CityC8ArrangementEngine.solve(...)`
- 得到：
  - `placements[]`
  - `success`
  - `errors`
  - `warnings`

这一步是当前 C8 的核心“坐标求解”逻辑。当前已经改成以 `area block / polygon` 集合做边界判定，不再使用 build area bbox 作为硬约束。

### 3. 地形统计怎么计算

`analyzeTerrain(...)` 会遍历当前 build area 覆盖到的所有 block：

- 统计：
  - `minH`
  - `maxH`
  - `avgH`
  - `p50`
  - `relief`
- 再调用 `estimateSlopeAvg(...)` 计算平均坡度
- 再额外计算四个边的平均高度：
  - `edgeN`
  - `edgeE`
  - `edgeS`
  - `edgeW`

### 4. foundation_type 怎么决策

当前规则是纯程序写死的：

- 如果 `relief <= 1 && slopeAvg < 0.4`
  - `foundation_type = NONE`
  - `strategy = FOLLOW_AVG`
- 如果 `relief >= 9`
  - `foundation_type = TERRACE`
  - `strategy = FOLLOW_P50`
- 如果边缘高差很强且 `relief >= 4`
  - `foundation_type = PLATFORM_WITH_RETAINING_WALL`
  - `strategy = CUT_AND_FILL`
- 否则
  - `foundation_type = PLATFORM`
  - `strategy = CUT_AND_FILL`

`baseY` 的规则：

- `FOLLOW_P50` 时用 `p50`
- 否则用 `avgH`

### 5. support 动作怎么来

如果 `foundation_type = PLATFORM_WITH_RETAINING_WALL`

- 自动补：
  - `retaining_wall`
  - `stairs`

如果 `foundation_type = TERRACE`

- 自动补一组 `stairs`

所以 C8 当前已经开始产出“地基策略”，不是只给 placements。

## 四、输入

- `city_id`
- `C6 summary/layout/index`
- 可选 `C7 selection`
- `height data`
- 可选 `C2 scan`

## 五、输出

- `C8_FoundationPlan.json`
- group 级 `c8_foundation.json`
- `c8_arrangement_debug.json`
- arrangement preview

输出字段重点：

- `foundation_type`
- `strategy`
- `base_y`
- `delta_height`
- `terrain_impact_extent`
- `terrain_metrics`
- `supports[]`
- `placements[]`
- `arrangement_success`
- `arrangement_errors`
- `arrangement_warnings`

## 六、AI 决策和程序计算的边界

### 1. AI 决策的部分

- 来自 C7：
  - `arrangement_type`
  - `arrangement_params`
  - `selected_template`
  - `component rules`

### 2. 程序计算的部分

- 生成具体 placements
- 计算地形统计
- 决定 foundation_type
- 决定 strategy / baseY / supports
- 保存 debug 文件

### 3. 当前代码和理想语义的差距

- 理想上 C8 只负责“执行排列算法”。
- 当前仍然夹带了一些 selection 补充字段和 fallback 兼容逻辑。
- 但主方向已经是程序按 polygon area 求坐标，AI 不再直接给最终 block 坐标。
