# C3.5 结构预处理与功能先验

## 当前状态

- 状态：部分实现
- 对应代码：`src/main/java/com/user/terra_script/world/city/stage/CityC35CatalogIO.java`
- 当前没有独立控制器和独立阶段入口，但 C7 已开始直接消费这层数据

## 阶段目标

- 在 C4 之前把结构级信息前置
- 建立结构特征库和功能总枚举表
- 为 C4/C7 提供统一的结构候选与语义约束

## 二、当前这层数据实际承担了什么职责

虽然没有独立的 `C3.5` 执行器，但当前 catalog 已经承载了下游需要的大部分结构先验：

- `function_enum_table`
- `structures[]`
- 每个 structure 的：
  - `size`
  - `orientation`
  - `function_candidates`
  - `connectors`
  - `placement`
  - `constraints`
  - `weight_profile`

也就是说，当前 C3.5 更像“静态结构契约层”，不是会主动跑一遍计算的程序阶段。
