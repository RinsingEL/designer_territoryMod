# C8 节点会话与提交实现

## 入口方法

| 位置 | 作用 |
| --- | --- |
| `CityController.handleCityC8Generate(...)` | 生成 `C8` 会话并按 group 过滤输出 |
| `CityController.handleCityC8Data(...)` | 读取当前 `C8` 会话 |
| `CityController.handleCityC8Submit(...)` | 提交节点决策；可选 `apply_now=true` 直接进入执行层 |
| `CityController.handleCityC8Retry(...)` | 重试失败节点 |

## 主要实现点

| 位置 | 说明 |
| --- | --- |
| `CityController.loadC8GenerationSelection(...)` | group 优先读取 `c7_selection.json`，没有才回退城市级 `C7` |
| `CityController.loadC9GenerationPlan(...)` | group 优先读取 `c8_foundation.json`，没有才回退城市级 `C8` |
| `CityC8Stages.generate(...)` | 从 `C6 + C7 + height + scan` 生成会话 |
| `CityC8Stages.submitNodeDecision(...)` | 校验模板、方向、旋转、坐标并写回 `placement` |
| `CityC8Stages.retryNode(...)` | 重置失败节点并回到会话队列 |
| `CityC8Stages.buildAreaGeometry(...)` | 根据 `C6` block index 生成 polygon 几何上下文 |

## 当前落盘路径

| 场景 | 路径 |
| --- | --- |
| 城市级 | `cityDir/C8_FoundationPlan.json` |
| group 级 | `groupDir/c8_foundation.json` |
| group 调试 | `CityC8Stages.saveDebug(...)` 输出到 group 目录 |

## 当前实现细节

- `handleCityC8Generate(...)` 在 group 模式下会同时导出 `arrangement_preview`。
- `handleCityC8Submit(...)` 会在同一条请求中完成：
  - 读取 `C8` 会话
  - 构建 `AreaGeometry`
  - 解析 `NodeDecision`
  - 调用 `submitNodeDecision(...)`
  - 回写同一份会话文件
- 当请求额外带 `apply_now=true` 时：
  - 当前节点仍先按 `C8` 规则完成 `validated`
  - 然后立即复用执行层管线尝试落地
  - 回包额外带 `placement_execution`
  - 失败时不回滚本次 `C8 validated` 结果，便于继续重试或调位
- `handleCityC8Retry(...)` 不重建整份计划，只在当前 `FoundationItem` 内调用 `retryNode(...)`。
