## W3 当前流程

1. 从 `ScanResultHolder.get().lastScanData` 读取全图扫描像素。
2. 调用 `ClusterAnalyzer.analyze(..., CONTINENT, ...)` 生成大陆区域。
3. 调用 `ClusterAnalyzer.analyze(..., OCEAN, ...)` 生成海洋区域。
4. 将大陆列表写入 `holder.lastClusters`，海洋列表写入 `holder.lastOceanRegions`。
5. 通过 `ClusterAnalyzer.expandOceans(...)` 生成 `holder.lastClusterMap`。
6. 导出 `world/W3/ContinentMeta.json` 与 `world/W3/OceanMeta.json`。

W3 没有外部请求体契约，核心前置是全局扫描缓存已经存在。

## W4 当前流程

1. 检查 `ScanResultHolder.get().regionCacheMap` 非空。
2. 遍历每个 region cache，汇总 `id / minX / minZ / w / h / step`。
3. 调用 `W4PreviewExporter.export(...)` 导出预览图索引。
4. 写出 `world/W4/TerrainSummary.json`。
5. 将 region 的 `slope / roughness / tpi` 矩阵压成 NBT long array。
6. 写出 `world/W4/TerrainFacts.dat`。

## MCP 侧入口

`country_designer_mcp/src/world/tools.ts` 暴露：

- `get_world_atlas`
- `world_scan_start`
- `world_scan_status`
- `world_scan_cancel`
- `world_w3_cluster`
- `get_t1_preview_maps`
- `world_w4_region_scan`
- `world_w4_region_status`
- `world_w4_export`
- `run_workflow_stage(stage_id="W3" | "W4" | "T1" | "T2" | "T3" | "T4")`
- `workflow_status`

其中 `run_workflow_stage` 会走 `/workflow/run`，适合执行阶段编排；独立的 W3/W4 工具适合调试和局部补缓存。

## 常见失败点

- W3 报 `W3 requires global scan data`：需要先跑 `/world/scan/start`。
- W4 报 `W4 requires region cache`：需要先对目标 region 跑 `/world/w4/region_scan`。
- T1 报 W4 数据缺失：通常是 `regionCacheMap` 没有目标大陆的 detail cache。
